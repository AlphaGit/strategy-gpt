- [x] 9.6 Implement node `rank` (score candidates by expected lift, evidence strength, complexity) — `nodes.py::rank_node(state)` sorts `state.accepted` desc by `rank_score(candidate) = 0.6 * estimated_lift_confidence + 0.3 * evidence_strength(kb_cites) - 0.1 * complexity_penalty(proposed_change)`. Complexity penalty is 1.0 when the proposed change carries any logic-change key (`source/code/rewrite/diff/patch/new_strategy`, mirroring the Tester's `parse_param_only_change` reject list) and 0.0 for param-only diffs. - [x] 9.8 Implement the inner iteration loop with three termination conditions (sufficient candidates / budget exhausted / similarity saturation) — `nodes.py::run_inner_loop(state, *, client, config, similarity_fn=None, now=None)` drives `generate → critique → rank` until a termination condition is met, then calls `select_node` with the decided reason. Termination priority: (1) `sufficient_candidates` once `len(accepted) >= target_candidates`, (2) `similarity_saturation` once every newly generated candidate this iteration scores `>= similarity_threshold` against pre-iteration rejections (in-loop `state.rejected` + `state.prior_decisions[REJECTED]`), (3) `budget_exhausted` once `iteration >= iteration_budget`. Saturation references are snapshotted *before* critique so the new batch is never compared against itself.
